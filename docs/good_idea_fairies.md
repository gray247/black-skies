- Items here **do not** imply commitment.
- No feature is allowed to jump straight from here into implementation:
  - It must first graduate into:
    - `docs/phases/phase_charter.md` (new phase or sub-phase), and
    - A proper spec (`docs/*.md`) if needed.
- This doc is allowed to be messy and imaginative, but must:
  - Name the idea
  - Specify rough value
  - Specify rough cost / risk
