# C5-1B — Readability Metric (Backend)
# Build the full readability composite metric using the helpers from C5-1A.
# No UI work. Backend only.

Read first:
  docs/phase6_analytics_refinement_notes.md   (C5-1B section)
  app/backend/services/analytics/text_utils.py
  app/backend/services/analytics/pipeline.py
  app/backend/services/analytics/*

Goal:
Implement a new backend readability metric that returns:

{
  "avg_sentence_len": float,
  "pct_long_sentences": float,
  "ttr": float,
  "bucket": "Easy" | "Moderate" | "Dense"
}

Rules:
1. Use ONLY helpers created in C5-1A:
      - sentence splitting
      - word tokenizing
      - long-sentence detection
      - TTR calculation
2. No UI, no endpoints, no renderer changes.
3. Integrate metric into analytics pipeline in a clean, isolated way:
      app/backend/services/analytics/pipeline.py
4. Do NOT calculate dialogue/narration or pacing here.
5. No HTTP calls. No remote services. Pure computation only.

Readability bucket rules:
- Easy:     avg_sentence_len < 14  AND pct_long_sentences < 0.10
- Moderate: avg_sentence_len between 14–20  OR pct_long_sentences between 0.10–0.20
- Dense:    avg_sentence_len > 20  OR pct_long_sentences > 0.20

Steps:

1. Create a new file:
       app/backend/services/analytics/readability.py
   Implement:
       compute_readability(text: str) -> dict
   This function:
       - splits text into sentences
       - tokenizes each sentence
       - computes:
             avg_sentence_len
             pct_long_sentences
             ttr
             bucket
       - returns full dict

2. Integrate readability metric into analytics pipeline:
       - modify scene-level analytics object
       - modify summary analytics object
       - ensure no breakage of existing keys

3. Update refinement notes:
       docs/phase6_analytics_refinement_notes.md
       Mark C5-1B as completed.

4. Update tracker:
       work/phase6/phase6_tracker.md
       Check off C5-1B.

5. (Optional but recommended)
   Add tests to:
       app/backend/tests/test_readability.py

At the end, print summary:
  - “Readability composite implemented”
  - “Readability pipeline integrated”
  - “Tracker + refinement notes updated”
