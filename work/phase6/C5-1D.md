# C5-1D — Pacing Metric (Backend Only)
# This task corresponds to C5-1D in:
#   docs/phase6_analytics_refinement_notes.md

# Objective:
# Implement the "structural pacing" metric using the helpers and metrics
# from C5-1A, C5-1B, and C5-1C.
# Backend only. No UI changes in this task.

Read first:
  docs/phase6_analytics_refinement_notes.md
  app/backend/services/analytics/text_utils.py
  app/backend/services/analytics/readability.py
  app/backend/services/analytics/pipeline.py
  app/backend/services/analytics/dialogue_narration.py (from C5-1C)

Requirements:

1) Inputs
   The pacing metric uses ONLY:
      - total scene length (characters OR tokens)
      - dialogue ratio from C5-1C

2) Computation
   Implement function:
       compute_pacing(text: str,
                      dialogue_ratio: float,
                      min_len_threshold: int = 200) -> dict

   Steps:
     - Compute scene_length = character count from text_utils.
     - Compute pacing_raw = (scene_length * (1 - dialogue_ratio))
     - Normalize if needed:
         If scene_length < min_len_threshold:
             classify as "Neutral" to prevent noisy extremes.

3) Pacing Buckets
   Bucket rules (override normalization only if text is long enough):

      If scene_length < min_len_threshold:
          bucket = "Neutral"
      Else:
          If pacing_raw < 2500:
              bucket = "Fast"
          If pacing_raw >= 2500 and pacing_raw <= 8000:
              bucket = "Neutral"
          If pacing_raw > 8000:
              bucket = "Slow"

   These thresholds are temporary and may be refined later.

4) Output structure
   Return:
       {
         "scene_length": int,
         "pacing_raw": float,
         "pacing_bucket": "Fast" | "Neutral" | "Slow"
       }

5) Integration
   - Add compute_pacing() into the analytics pipeline:
         app/backend/services/analytics/pipeline.py
   - Require dialogue_ratio from C5-1C.
   - Add pacing results to:
         /analytics/scenes
         /analytics/summary

6) Error Handling
   - If text is empty → length = 0, pacing_raw = 0, bucket = "Neutral".
   - If dialogue_ratio missing → treat as 0 (fully narration).
   - All errors must fail gracefully.

7) Documentation
   - Update docs/phase6_analytics_refinement_notes.md:
         Mark C5-1D completed.
   - Update work/phase6/phase6_tracker.md:
         Check off C5-1D.
   - Add note to work/phase6/verification_notes.md.

8) Tests (optional but recommended)
   Create:
      app/backend/tests/test_pacing.py

When finished, print:
  - "Pacing metric implemented"
  - "Pacing pipeline integrated"
  - "Tracker + refinement notes updated"
